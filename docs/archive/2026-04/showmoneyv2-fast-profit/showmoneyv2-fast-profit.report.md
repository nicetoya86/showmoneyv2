## 6. 주요 학습 사항

### 6.1 설계 인사이트

1. **필터 순서가 결과를 결정한다**: RSI 필터를 RVOL 계산 앞에 두면 급등 후보 판별 자체가 불가능. 필터 순서 재배치가 급등 포착의 핵심이었다.

2. **HIGH IMPACT 지표 조합 원칙**: 단일 지표(RVOL)보다 복수 지표 이중 확인(RVOL + vol_trend_5_60)이 "우연한 급등"과 "수급이 쌓이는 급등"을 구분한다.

3. **발송 정책 우선**: 포착 정밀도 개선보다 **노이즈 차단**이 사용자 경험에 즉각적 영향. `MIN_DAILY_PICKS=0` 한 줄이 2주 대기 알림을 완전 제거한다.

### 6.2 구현 패턴

- **`isSurgeCandidate` 플래그 패턴**: 여러 필터에서 동일 조건을 반복 검사할 때 Boolean 플래그로 사전 계산 → 가독성 + 성능 개선
- **`store.sectorMap` 지속성 활용**: n8n workflow의 `store` 객체를 업종 맵으로 활용하면 실행 간 데이터 유지 가능
- **`FAST_GRADES Set` 패턴**: 발송 허용 등급을 Set으로 관리하면 등급 추가/제거가 상수 변경만으로 가능
