## 1. 4월 18일 리포트 실전 검증 (신규 추가)

### 1.1 문제 현황

4월 18일 주간 리포트에서 **추천 종목은 많으나 ✅ 수익(목표가 도달) 종목이 매우 적은** 현상 확인.

**구조적 원인 5가지:**

| # | 원인 | 코드 위치 | 상세 |
|---|------|-----------|------|
| R-01 | 목표가 설정 지나치게 공격적 | `ATR_TARGET_MULT = 2.8` | 손절(×1.9) 대비 목표(×2.8) — 강한 추세 없으면 도달 불가 |
| R-02 | 급등/매도차익 보유기간 2일 너무 짧음 | `HOLD_SURGE = 2` / `HOLD_SHORTTRADE = 2` | 2일 내 목표 도달 확률 낮음 |
| R-03 | OBV 수급 확인이 점수 보너스에 불과 | `score += 20` (수급확인) | 실제 매수세 없는 종목이 패턴 점수만으로 통과 |
| R-04 | 신고가 근접 과대평가 | `pth≥0.95 → +25점` | 저항선 위 물량 있음에도 고점수 획득 |
| R-05 | 매수 진입이 하락 중 저가 터치 방식 | `lowPrice <= r.entry` 진입 판정 | 저가 터치 = 매도 압력 존재, 이후 추가 하락 가능 |

### 1.2 목표가 도달 vs 미도달 종목 특성 비교

| 특성 | ✅ 목표가 도달 | ❌ 손절/미도달 |
|------|---------------|----------------|
| 핵심 신호 | 52주 신고가 **돌파** + OBV 수급 + RVOL A등급 | 패턴점수 위주 (신고가 **근접**만) |
| 등급 | 강매(5일 보유) 多 | 급등/매도차익(2일 보유) 多 |
| OBV 방향 | 상승 (수급확인) | 중립/하락 (수급불일치) |
| ATR 특성 | 낮은 ATR → 목표가 상대적으로 가까움 | 높은 ATR → 목표가 너무 멀리 |

## 4. 갭 분석 (데이터 + 4월 18일 실전 검증)

### GAP-01: 금요일 진입 패널티 (이미 구현됨)

- **현재 코드:** `DOW_PENALTY_FRI = 5` (rankScore -5점)
- **상태:** ✅ 구현완료

### GAP-02: 요일별 보너스 (이미 구현됨)

- **현재 코드:** 목요일 +3, 수요일 +2, 금요일 -5
- **상태:** ✅ 구현완료

### GAP-03: 목표가 배수 과도 ← 4월 18일에서 확인된 핵심 문제

- **데이터:** 최다 수익 구간 5~10% (41.6%)
- **현재:** ATR × 2.8 → 실제 목표가 10~15% 수준으로 설정
- **문제:** 강한 추세 없이는 도달 불가, 목표가 도달률 낮음
- **개선안:** ATR × 2.0 (기본) / 강매등급 시 ATR × 2.8 유지

### GAP-04: OBV 수급 확인이 필수 조건 아님

- **현재:** `score += 20` (선택적 보너스)
- **문제:** 실제 매수세 없는 종목도 패턴 점수만으로 통과
- **개선안:** OBV 수급 확인(obvTrend === 1) OR RVOL A등급(3x+) 중 하나 필수 조건화

### GAP-05: 신고가 근접(PTH) 과대평가

- **현재:** `pth≥0.95 → +25점`, `pth≥0.90 → +15점`
- **문제:** 신고가를 뚫지 않은 종목 = 저항선 위 물량 대기 → 목표 도달 어려움
- **신고가 돌파(+40)와 격차 너무 적음**
- **개선안:** `pth≥0.95 → +15점`, `pth≥0.90 → +8점` (하향 조정)

### GAP-06: 급등/매도차익 보유기간 2일 부족

- **현재:** HOLD_SURGE = 2, HOLD_SHORTTRADE = 2
- **문제:** 2일 내 목표가 ATR×2.8 도달은 강한 종목 외 거의 불가
- **개선안:** HOLD_SURGE = 3, HOLD_SHORTTRADE = 3 (1일 연장)

### GAP-07: 목표가 상한 캡 (기존 갭 유지)

- **현재:** `CAP_TARGET_PCT = 0.25` (25%)
- **데이터:** 20%+ 케이스 15.9%
- **상태:** 이미 25%로 충분 — 추가 조정 불필요

### 5.2 ATR 목표 배수 변경 상세

```javascript
// 현재
const ATR_TARGET_MULT = 2.8;

// 개선안
const ATR_TARGET_MULT_NORMAL = 2.0;   // 기본 (급등, 매도차익, 완화)
const ATR_TARGET_MULT_STRONG = 2.8;   // 강매 등급 (5일 보유, 충분한 시간)

// 적용 로직
const targetMult = (grade === '강매') ? ATR_TARGET_MULT_STRONG
                 : (grade === '매도차익') ? ATR_TARGET_SHORT  // 기존 1.5x 유지
                 : ATR_TARGET_MULT_NORMAL;
```

### 5.3 OBV 필수 조건화 상세

```javascript
// 현재: 점수 보너스만
if (obvResult.obvTrend === 1) score += 20;

// 개선안: 필수 조건 (OBV 확인 OR RVOL A등급 중 하나)
const hasSupply = (obvResult.obvTrend === 1) || (rvolVal >= RVOL_GRADE_A);
if (!hasSupply && grade !== '강매') return; // 수급 미확인 & 강매 아님 → 차단

// OBV 점수는 기존 유지 (+20)
if (obvResult.obvTrend === 1) score += 20;
```
