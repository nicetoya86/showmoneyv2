| **Problem** | 고정 손절가(ATR 1.9x)는 진입 시점에 고정되어 이후 가격 상승 이익을 전혀 보호하지 못함. 남선알미늄(+16.1% 고점)처럼 큰 수익 기회가 있었던 종목도 시장 급락으로 원래 손절가에 도달해 '손절'로 기록됨 |

| **Core Value** | 진입 신호 품질보다 **포지션 관리**가 성과에 더 큰 영향을 미친다는 실증 데이터 기반의 시스템 개선 |

## 2. 근본 원인 분석

### 2.1 현재 손절 메커니즘의 구조적 결함

```
현재 구조:
  진입가: X
  손절가: X - ATR * 1.9 ≈ X * 0.90 (고정, 진입 후 불변)

문제 시나리오 (남선알미늄):
  진입가: X
  +16.1% 도달: X * 1.161 (최고점)
  매크로 급락: X * 1.161 → X * 0.90 (고정 손절가 도달)
  결과: "손절" 기록 (실제론 +16.1% 기회가 있었음)

트레일링 스탑이 있었다면:
  +5% 도달 → 손절을 X * 1.00 (브레이크이븐)으로 올림
  +10% 도달 → 손절을 X * 1.05로 올림
  최고 +16.1% 후 급락 → X * 1.05 또는 X * 1.08에서 청산
  결과: "이익 실현" 기록
```

### 2.2 주간 리포터 평가 로직의 오분류

- 현재: `low <= r.stop` 이면 무조건 '손절(loss)'
- 문제: 고점 대비 되돌림을 고려하지 않음
- 개선: 트레일링 스탑 시뮬레이션으로 실제 청산가 계산

### 2.3 매크로 시장 충격 무방비

- 4/2 케이엔더블유, 컴퍼니케이 동시 손절 = 종목 선택이 아닌 시장 전체 문제
- 미국 관세 쇼크("Liberation Day") 등 시장 급락 시 진입 차단 메커니즘 없음
- KOSPI/KOSDAQ 지수 동향 필터 부재

## 4. 상세 개선안

### 4.1 TRAIL-01 — 트레일링 스탑 포지션 모니터 (신규 파일)

**개요:** n8n에서 매일 장 마감 후 실행되는 포지션 관리 노드. 활성 포지션의 현재가를 조회하여 트레일링 스탑/브레이크이븐을 적용하고 Telegram으로 업데이트된 손절가를 안내.

**트레일링 스탑 단계:**

| 조건 | 현재 손절가 조정 | 기대 효과 |
|------|--------------|---------|
| 수익률 < +3% | 원래 손절가 유지 | 변동 없음 |
| 수익률 ≥ +3% | 손절가 → 진입가 (브레이크이븐) | 원금 보호 |
| 수익률 ≥ +5% | 손절가 → 최고가 - ATR * 1.5 | 이익 일부 잠금 |
| 수익률 ≥ +10% | 손절가 → 최고가 - ATR * 1.0 | 이익 대부분 잠금 |
| 수익률 ≥ +15% | 손절가 → 최고가 - ATR * 0.7 | 이익 최대 보호 |

**데이터 흐름:**
```
매일 장 마감 (15:30 KST):
  1. store.weeklyRecommendations에서 활성 포지션 로드
  2. 각 종목 당일 고가/저가/종가 조회 (Naver Finance API)
  3. 트레일링 스탑 로직 적용 → 새 손절가 계산
  4. store.weeklyRecommendations 손절가 업데이트
  5. 변경된 종목만 Telegram 알림 발송
```

**Telegram 알림 예시:**
```
📊 [포지션 업데이트]
남선알미늄(008350)
  현재가: 21,450원 (+8.3%)
  ✅ 손절가 상향: 18,500원 → 20,200원 (브레이크이븐+3%)
  만료: 2026-04-08 (3일 남음)
```

### 4.2 REPORT-01 — 주간 리포터 트레일링 스탑 시뮬레이션

**개요:** 주간 리포터가 성과 평가 시 원래 고정 손절가가 아닌, 트레일링 스탑 로직을 시뮬레이션한 **동적 손절가**로 결과를 판정.

**평가 로직 변경:**

```
현재:
  if (low <= r.stop) → hitStopDay 기록 → 'loss'

변경:
  // 트레일링 스탑 시뮬레이션
  let trailingStop = r.stop; // 초기값 = 원래 손절가
  let maxHigh = r.entry;

  for (const day of relevant) {
    // 최고가 업데이트
    if (day.high > maxHigh) {
      maxHigh = day.high;
      const gain = (maxHigh - r.entry) / r.entry;

      // 트레일링 스탑 단계적 상향
      if (gain >= 0.15 && r.atrAbs) {
        trailingStop = Math.max(trailingStop, maxHigh - r.atrAbs * 0.7);
      } else if (gain >= 0.10 && r.atrAbs) {
        trailingStop = Math.max(trailingStop, maxHigh - r.atrAbs * 1.0);
      } else if (gain >= 0.05 && r.atrAbs) {
        trailingStop = Math.max(trailingStop, maxHigh - r.atrAbs * 1.5);
      } else if (gain >= 0.03) {
        trailingStop = Math.max(trailingStop, r.entry); // 브레이크이븐
      }
    }

    // 트레일링 스탑 도달 여부 체크
    if (!hitStopDay && day.low <= trailingStop) {
      hitStopDay = ...;
      trailStopPrice = trailingStop; // 실제 청산가 기록
    }
  }
```

> **이 개선으로:** 남선알미늄 +16.1% 후 손절 → **이익 청산(+8% 이상)**으로 재분류

### 4.3 REGIME-01 — 시장 Regime 필터

**개요:** 스윙 스캐너 실행 시 KOSPI 지수를 조회하여 시장 전체가 하락 추세인 경우 추천 기준을 강화하거나 진입을 차단.

**Regime 판단 기준:**

| Regime | 조건 | 대응 |
|--------|------|------|
| 강세 | KOSPI 5일 이평 > 20일 이평 | 기본 기준 유지 |
| 중립 | KOSPI 5일 < 20일 이평, 하락 < -3% | PMAT 기준 +10%p 상향 |
| 약세 | KOSPI 당일 -2% 이상 하락 | MAX_RECS 50% 감소 |
| 급락 | KOSPI 당일 -3% 이상 하락 | 당일 추천 전면 중단 |

**데이터 소스:** 네이버 금융 KOSPI 지수 API (`서울증권시장:KOSPI`)

### 4.4 ENTRY-01 — 당일 변동성 과다 진입 필터

**개요:** 당일 종목의 고저 범위(Range)가 ATR 대비 과도하면 진입 스킵. 4/3 애경케미칼처럼 관세 쇼크 당일 급변동 종목 진입 차단.

**조건:** `(high - low) > ATR * 3.0` → 해당 종목 스킵 (`'변동성 과다'` 로그)
